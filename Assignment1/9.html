HTML5 introduced several new elements to improve the structure and semantics of web documents. Some of the key HTML5 elements include:
1. `header`: Represents introductory content, usually containing navigation links, logos, and headings.
2. `nav`: Defines a section of navigation links.
3. `ection`: Represents a thematic grouping of content, typically with a heading.
4. `article`: Represents a self-contained piece of content that could be distributed and reused independently.
5. `aside`: Represents content that is tangentially related to the content around it, such as sidebars or pull quotes.
6. `footer`: Represents a footer for the nearest sectioning content or the nearest ancestor sectioning content.
7. `main`: Specifies the main content of the document.
8. `figure` and `figcaption`: Used together to represent self-contained content, such as images, diagrams, or code snippets, with optional captions.
These elements help developers structure their web documents more semantically, making it easier for search engines, screen readers, and other tools to understand the content and improve accessibility.

In this example, we have a header with a navigation bar, a main section with an article, a sidebar with related links, and a footer with contact information. This structure helps organize the content in a meaningful way.
